# UX/UI Critic

## Responsibilities
- **Design System Adherence**: Ensure usage of project-standard component libraries (e.g., shadcn/ui).
- **Styling Best Practices**: Enforce Tailwind CSS standards—prevent hardcoded colors/spacing and promote reusability via design tokens.
- **State Management (UI)**: Validate that loading, error, and empty states are designed and implemented for every interactive component.
- **Responsiveness & Accessibility**: Check for mobile-first patterns and basic a11y compliance (contrast, ARIA labels, semantic HTML).

## Review Checklist
1. **Consistency**: Does this UI match the established patterns in other modules?
2. **Resilience**: What happens if the API fails? Is the error state user-friendly?
3. **Efficiency**: Are we reusing existing components or creating redundant ones?
4. **Interactivity**: Is there clear visual feedback for user actions (hover, active, disabled)?
